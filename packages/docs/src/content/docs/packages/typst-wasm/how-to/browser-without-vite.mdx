---
title: Use the browser without Vite
description: Load typst-wasm assets explicitly in a bundler-neutral browser application.
---

import { Aside, Steps } from "@astrojs/starlight/components";

This is an advanced setup. Unlike the Vite guide, you must publish the worker, engine, WASM, and font assets yourself and provide their URLs to the compiler.

<Aside type="caution" title="Prefer Vite when possible">
  Vite's `?worker&url` and `?url` imports handle asset copying and URL
  generation automatically. Use this guide only when your build system has an
  equivalent asset pipeline.
</Aside>

<Steps>

1. **Install and copy runtime assets**

   Install `typst-wasm` and `@typst-wasm/fonts`, then copy these package assets into your public asset directory:

   - `typst-wasm/worker/web-worker`
   - the engine module and its core WASM files
   - the fonts your documents require

2. **Create a module worker**

   ```ts
   import { createTypstCompiler, createWebWorker } from "typst-wasm/browser";
   const workerUrl = "/assets/typst-wasm/web-worker.js";
   const compiler = await createTypstCompiler({
     backend: "auto",
     worker: () => createWebWorker(workerUrl),
     coreModules: {
       "engine.core.wasm": WebAssembly.compileStreaming(
         fetch("/assets/typst-wasm/engine.core.wasm"),
       ),
       "engine.core2.wasm": WebAssembly.compileStreaming(
         fetch("/assets/typst-wasm/engine.core2.wasm"),
       ),
       "engine.core3.wasm": WebAssembly.compileStreaming(
         fetch("/assets/typst-wasm/engine.core3.wasm"),
       ),
     },
   });
   ```

3. **Load fonts by URL**

   ```ts
   const response = await fetch("/assets/fonts/NewCMMath-Regular.otf");
   await compiler.addFonts(new Uint8Array(await response.arrayBuffer()));
   ```

4. **Compile and dispose**

   ```ts
   await compiler.addSource("main.typ", "= Hello from Typst");
   const result = await compiler.compile({
     main: "main.typ",
     format: "html",
   });
   document.querySelector("#output")!.innerHTML = result.output;
   await compiler.dispose();
   ```

</Steps>

The exact engine and worker asset layout depends on your bundler. The browser page must also satisfy the [browser deployment requirements](../../deployment/browser-requirements/).
