An isomorphic application can compile the same document in two runtimes:

- the server compiles the first document while rendering the route, so the response already contains a preview;
- after hydration, the browser compiles edits in a Web Worker without waiting for the server.

This page walks through the pattern used by the [TanStack Start playground](https://github.com/typst-wasm/typst-wasm/tree/main/packages/example-typst-playground-cloudflare) on Cloudflare. The pattern is not specific to TanStack Start or React, but it requires framework specific code.

## Creating an isomorphic function

Creation of our compiler is going to work slightly differently on the server vs client. On the server, we don't have access to any workers, so we must use the JSPI backend.

TanStack Start makes isomorphic behaviour easy with `createIsomorphicFn`, but you could do this with other frameworks using something like Vite's `import.meta.env.SSR`.
